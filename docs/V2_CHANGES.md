# v2 變更說明（回應審查回饋）

針對組長 / 審查回饋的 8 點，逐項修正如下。前端 UI 文字為繁體中文，後端 API 不變動既有路徑、只新增。

| # | 回饋 | 處理方式 |
|---|------|---------|
| 1 | 系統管理員不該能審查，只有審查單位能審查 | RAS 兩支 API 改為 **僅 REVIEWER**；前端審查頁與首頁入口也只對 REVIEWER 開放。ADMIN 不再出現「審查申請」。 |
| 2 | 管理員 CRUD 只有「新增」，缺修改/刪除（帳號與獎學金都是） | 新增 **PUT/DELETE /api/aas/users/{id}** 與 **PUT/DELETE /api/sms/scholarships/{id}**；前端帳號管理頁可逐筆「編輯/刪除」，獎學金管理頁每張卡片可「編輯/刪除」（含關閉）。刪除若有關聯資料會擋下並提示。 |
| 3 | 學生已申請過的獎學金，申請鈕仍可點 | 申請頁載入時會同時抓「我的申請」，已申請的項目按鈕改為停用並顯示「已申請」。 |
| 4 | 申請應該有「申請表」，不是隨便打字 | 申請改為結構化表單：申請理由、聯絡電話、通訊地址、家庭狀況、在學成績/排名；頁面上方顯示唯讀的申請人身分（姓名/學號/系所/GPA）。 |
| 5 | 推薦信系統、通知/溝通系統完全沒有 | **TRS**：學生在「我的申請」邀請老師→老師在「推薦邀請」頁撰寫/存草稿/送出（學生只看得到狀態，看不到內容）。**NCS**：站內通知（申請送出、審查結果、推薦邀請/送出都會通知）+ 公告（管理員可發布），導覽列有未讀鈴鐺。 |
| 6 | 審核沒有 log，審查人員/結果沒被記錄 | 每次審查寫入 `reviews`（審查人、結果、意見、時間）；審查清單會顯示「最近審查紀錄」。另新增 `audit_logs` 稽核表，記錄帳號/獎學金的新增修改刪除，管理員可在「稽核紀錄」頁檢視。 |
| 7 | 學生無法修改自己的個人資料 | 新增 **GET/PUT /api/sas/profile** 與「個人資料」頁：聯絡電話/地址/緊急聯絡人可改；學號/姓名/GPA 等核心資料唯讀（依需求 6.2.1）。 |
| 8 | 頁面太簡陋，需要更多前端 | 新增全站頂部導覽列（依角色顯示功能、未讀通知鈴鐺、登入者/登出）、未登入自動導向登入頁的全域守衛、首頁改為角色化主控台，並統一卡片式版型與配色。 |

## 資料庫注意事項（重要）
v2 動到資料表結構：
- 新表：`audit_logs`、`student_profiles`、`recommendations`、`notifications`、`announcements`（用 `CREATE TABLE IF NOT EXISTS`，全新建立沒問題）。
- `applications` 多了 5 個欄位；`03_sas_tables.sql` 內含「若欄位不存在才補上」的相容語句，**舊資料庫也會自動補欄位**。

若你之前**已經跑過 v1 的 `start.bat`**（資料庫已存在），直接再跑一次即可，腳本會自動補表/補欄位。萬一遇到欄位相關錯誤，最保險是把資料庫整個重建一次：在 MySQL 執行 `DROP DATABASE nuksams;` 後再跑 `start.bat`（會重新建立 + 重新載入種子資料）。

## 種子帳號（demo）
沿用 v1：跑過一次後執行 `.venv\Scripts\python -m app.modules.aas.dev_seed`，密碼皆 `password123`：
- `admin`（系統管理員）、`sponsor`（獎助單位）、`reviewer`（審查人員）、`teacher`（老師：陳老師）
- `A1125529`（學生，GPA 3.85）、`A1125599`（學生，GPA 3.20）

完整流程可測：以 `A1125529` 登入→申請獎學金→在「我的申請」邀請「陳老師」→改用 `teacher` 登入撰寫並送出推薦信→再用 `reviewer` 登入審查（可看到推薦信內容）→`A1125529` 會收到審查結果通知。
