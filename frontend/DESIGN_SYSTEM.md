# NUKSAMS 視覺識別 — 「印 / The Conferral」

獎助學金系統的核心，是一個古老的制度行為：學術單位**核定**一筆獎助、並**留存紀錄**。
在臺灣公文體系裡，這個行為由一枚紅色印信驗證。整套介面因此被裝扮成
**一份裝在牛皮卷宗裡的正式紀錄**——牛皮紙當外框、墨色當結構、白色內頁承載工作畫面，
唯一的簽名元素是一枚**硃砂印**。

> 本識別是**疊加在既有的真實後端串接前端（v2）之上**：所有頁面的 `<script>`
> 後端邏輯（登入、API 呼叫、路由）完全未動，只重做視覺層。

## 架構

- **全域設計系統** `src/assets/styles.css`（由 `main.js` 匯入）：色彩 token、字體、基礎元件樣式、`.seal` 印章。改色只需改 `:root`。
- **各頁面**沿用既有的 scoped `<style>` 慣例，把寫死的藍灰色**改指向 token**（例：`#2b6cb0` → `var(--primary)`）。
- 字型在 `index.html` 以 Google Fonts 載入。

## 色彩（token → 用途）

| Token | 值 | 角色 |
| --- | --- | --- |
| `--text` | `#23201a` | 暖墨黑：文字與結構線 |
| `--paper` | `#efe7d4` | 牛皮卷宗：工作區底色 |
| `--surface` | `#fdfbf6` | 白色內頁：卡片、頂欄、表格 |
| `--primary` (jade 青) | `#3d6a55` | **所有操作**：按鈕、連結、作用中導覽、焦點 |
| `--seal` (cinnabar 硃砂) | `#bb3b2a` | **簽名色**：品牌印記、登入浮水印。**不當一般按鈕用** |
| `--gold` (brass 赭金) | `#8a6a2c` | 金額／榮譽／補件提醒 |
| `--success` | `#46774e` | 通過／啟用 |
| `--info` | `#3a5f86` | 審核中（鋼藍）|
| `--danger` | `#9a3325` | 退件／刪除（暗赭紅，刻意比印記更深沉以免混淆）|

設計上的一個取捨：**主色用綠（青）而非藍**。行政系統極少用綠色主色，一眼就跟
「SaaS 後台藍」拉開距離；同時把暖紅留給典禮性的印記時刻，紅色就不必兼任「錯誤」。

## 字體（三種角色）

- `--font-serif` = **Spectral + Noto Serif TC** — 標題與大數字。宋體正是公文與典籍的字體。
- `--font-sans` = **Inter + Noto Sans TC** — 密集 UI（表格、表單）主力。
- `--font-mono` = **IBM Plex Mono** — 學號、申請編號、日期等「編號」性質的參照碼。

## 簽名元素：硃砂印 `.seal`

紅底白字、內嵌白色雙線框，模擬公文印信。

1. `.brand__seal`（頂部導覽列）— 「奬」字印。
2. `.seal--watermark`（登入頁）— 大型低透明印記，旋轉 -4°，溢出邊角，是頁面的記憶點。

把大膽集中在這一個元素，其餘保持安靜——這是刻意的克制。

## 品質底線

- 鍵盤焦點：全域 `:focus-visible` 青色描邊。
- `prefers-reduced-motion`：關閉動畫與轉場。
- 表格／表單元件由全域樣式統一，各頁不再各自硬刻。
